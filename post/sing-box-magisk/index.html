传送阵#

传送阵套娃：包含其他传送阵。你的默认只有 direct（直连，不通过代理）、block（断开，“黑洞路由”）、proxy（代理）、domestic（国内）、GLOBAL：全局模式下流量代理节点

  1. 新增名为"GLOBAL"的传送阵
  2. 修改传送阵"proxy"的"传送阵套娃"选择"GLOBAL"
  3. 传送阵"domestic"国内代理选择"百度直连"(免流佬专用)
  4. 新增"ChatGPT"的传送阵，传送阵套娃选择"proxy",星盘选择MJJ_ChatGPT，我想要这些：((gpt).*(0.5))|((0.5).*(gpt)) 正则表达式，表示搜索同时包含关键字gpt和0.5筛选节点
  5. 进入神秘，勾选热点模式、兼容模式

打开 /data/adb/sfm/src/baseConfig.hjson 文件，修改

怎么放行应用？#

修改 packages_list 里的 black 或 white，它们分别是黑名单（放行列表内的应用）和白名单（只代理列表内的应用）。

mode 默认为黑名单，无须修改，如果要修改，参考白名单。在 black 中添加包名，一行一个。例如旅行QQ、微信

  packages_list:
  {
    enabled: true
    mode: black
    black:
    [
      com.tencent.mobileqq
      com.tencent.mm
    ]

  black:
    - com.tencent.mobileqq
    - com.tencent.tim
    - com.tencent.mm
    - com.taobao.taobao
    - com.eg.android.AlipayGphone
    - com.taobao.idlefish
    - com.ss.android.ugc.aweme
  users:
    - 0
    - 999

可以使用半成品面板，点击动图进入设置 - 名单里给对应的应用启用即可。

list: [
  {
    package_name: com.tencent.mm
    users: [
      0
      999
      10
    ]
  }
]

如果你的手机有多个分身，可以通过 su -c pm list users 获取所有分身的 user_id，只需要记住主空间一般是 0，双开应用一般是 999，手机分身一般是 10。

  1. interface_name：网卡名，一般情况请勿修改。
  2. inet4_address/inet6_address：tun接口前缀，请勿修改。
  3. mtu：最大传输单元，此项在热点模式下可以根据情况修改至10000以下。一般情况下默认即可。
  4. stack：TCP/IP 栈，system/gVisor均可，一般情况下无需修改
  5. exclude_package：排除路由应用，即黑名单模式，在列表内的应用流量不进入tun
  6. include_package：限制路由应用，只有列表内提及到的应用包名流量进入tun，其余软件不代理，即白名单模式
  7. strategy：解析策略，节点支持ipv6的情况下可更改为prefer_ipv4/prefer_ipv6，但更推荐使用prefer_ipv4或ipv4_only。

路由规则#

路由规则是一个非常强大的功能，能够实现不同站点、地区、应用之间的分流操作，同时可以实现屏蔽某些连接或是对特定网络访问进行本地放行。

语法示范说明

rules:[
      {
        protocol: 'dns',
        outbound: 'dns',
      },
  ]

Geosite为路由规则教程介绍中讲解的第一个字段，这要归功于其方便快捷且简单的使用方法，以及可以自动更新的即时性

      {
        geosite: [
          'boc',
        ],
        outbound: 'direct'
        /* outbound为指定出站，即分流至哪一节点or分组 */
       },

domain（完整域名）、domain_suffix（域名后缀）、domain_keyword（域名关键词）、domain_regex（域名正则）此类规则的使用较为简单，本段内容以domain_keyword字段为例介绍此类规则的使用方法

      {
        domain_keyword: [
          '3debspay',
        ],
        outbound: 'direct',
      }
  1. Geoip规则：匹配Geoip，使用方法与Geosite相似，在此不做过多解释
  2. ip_cidr：匹配cidr段
  3. ip_version：ipv4或ipv6（如需使用ipv6请注意baseconfig注释，删除屏蔽ipv6的路由规则）
      {
        geoip: [
          'cn',
        ],
        outbound: 'domestic',
      },
      {
        ip_cidr: [
          '20.222.0.0/16',
        ],
        outbound: 'proxy',
      },
      {
        ip_version: 6,
        outbound: 'block',
        /* block为屏蔽该类型连接 */
      },

port：

  1. port：匹配某一端口
  2. port_range：匹配端口范围

（该类规则一般与domain规则以及ip_cidr共同使用）

      {
        port: [
          '7100',
        ],
        outbound: 'direct',
      },
      {
        port_range: [
          "1000:2000", /* 1000始，2000终 */
          ":3000", /* 0始，终止至3000 */
          "4000:" /* 起始于4000，终止于65535 */
        ],
        outbound: 'direct',
      },

package_name：

匹配安卓包名，包括主进程以及子进程

      {
        package_name: [
          'com.netflix.mediaclient'
        ],
        outbound: 'proxy'
      },

protocol

协议探测，如http、tls、quic、stun等

      {
        protocol: [
          'stun',
        ],
        outbound: 'block'
       },

与筛星盘共同使用，定向分流方法

本次以Netflix举例，将Netflix应用流量分流至新加坡解锁节点

  1. 设置星盘司南输出目标：输出到上级分组，建议此类分流节点不输出。
  2. 正向筛法：正则表达式筛选符合条件节点，例：((新加坡).*(流媒体))|((流媒体).*(新加坡)) 既包含“新加坡”又包含“流媒体”。
  3. 反向筛法：正则表达式筛选不符合条件的节点
  4. 选择要筛的星盘：筛选订阅节点分组来源
  5. 另加传送阵：另外添加的节点

路由规则举例：

      {
        package_name: [
          'com.netflix.mediaclient'
        ],
        outbound: 'Netflix'
      },

逻辑字段

为路由规则设定另外的匹配逻辑，即不使用默认匹配逻辑

  1. type：默认logical即可
  2. mode：and或是or，and表示多条规则同时符合才按此条规则分流，or表示符合其中一条规则即按此条规则分流
  3. rules：与route.rules一致，在此字段内的子规则之间的关系不使用默认匹配逻辑
  4. invert：反选匹配结果
    rules: [
      {
        type: 'logical',
        mode: 'and',
        rules: [
          {
            domain_keyword: [
              'boc',
            ],
          },
          {
            domain_keyword: [
              '3debspay',
            ],
          },
        ],
        invert: false,
        outbound: 'direct'
      },
    ],

多种路由匹配规则条件同时符合

本次将以游戏プロセカ为例，举例多条匹配字段同时符合规则的写法，プロセカ的多人游戏需要udp，而多数节点并无udp转发功能，且屏蔽idc ip，因此该游戏多人游戏的udp连接需要对特定端口放行直连。

首先将逻辑模式设置为and，再通过包名匹配匹配到游戏进程，其次匹配此进程所有udp连接的7100端口，并输出到direct。

      {
        type: 'logical',
        mode: 'and',
        rules: [
          {
            package_name: [
              'com.sega.pjsekai'
            ],
          },
          {
            network: 'udp',
            port: [
              7100,
            ],
          },
        ],
        outbound: 'direct',
      },

override_android_vpn：

VPN 默认优先于 tun。要使 tun 经过 VPN，启用此项时，接受 Android VPN 作为上游网卡。

  {
    "route": {
      "geoip": {},
      "geosite": {},
      "rules": [],
      "final": "",
      "override_android_vpn": true,
    }
  },

使用规则文件添加路由，比如ChatGPT分流

    - tag: ChatGPT-域名
      path: ./RuleProviders/ChatGPT-域名.srs
      url: >-
        https://raw.githubusercontent.com/CHIZI-0618/v2ray-rules-dat/release/singbox_rule_set/geosite-openai.srs
      type: remote
      format: binary
      download_detour: 国外出口
      update_interval: 24h

在rules字段中添加

    - rule_set:
        - ChatGPT-域名
      outbound: ChatGPT
